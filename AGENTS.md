# AGENTS.md ‚Äî NixOS Configuration Repository

## What This Is

NixOS configuration managed as a GitOps flake. Targets two laptops:
- **p1g3** ‚Äî Lenovo ThinkPad P1 Gen 3 (Intel + NVIDIA Optimus)
- **t14s** ‚Äî Lenovo ThinkPad T14s Gen 1 (Intel i7 10th gen, no dGPU)

Both share ~90% of config (Hyprland, packages, services, theme).

## Current State ‚Üí Target

**Current (pre-refactor):** Flat structure ‚Äî `flake.nix`, `configuration.nix`, `hardware-configuration.nix`, `home.nix` at root. Single host.

**Target (post-refactor):** Dendritic pattern ‚Äî `flake-parts` + `import-tree`. Every `.nix` under `modules/features/` is an auto-discovered module. Hosts are thin assembly files.

**Read first:**
- `docs/prd/dendritic-refactor.md` ‚Äî full PRD with all design decisions and migration phases
- `docs/issues/` ‚Äî numbered issues tracking each incremental step

## Critical Rules (Would an Agent Miss These?)

### `useGlobalPkgs = true` ‚Äî Overlay Scope

Overlays defined in Home Manager modules are **silently ignored**. Define overlays in the host's NixOS config block (`home-manager.nixpkgs.overlays`), never in a `.nix` file inside `modules/features/` that registers as `homeModules`.

### `hardware-configuration.nix` ‚Äî Never Edit

These files are generated by `nixos-generate-config`. Custom hardware config belongs in feature modules, not in these files. Regenerate by running `sudo nixos-generate-config --no-filesystem --dir <host-dir>/`.

### Two Nixpkgs Inputs

- `nixpkgs` (nixos-26.05) ‚Äî stable, primary package set
- `nixpkgs-unstable` ‚Äî for select packages (vscode, joplin, ty, uv, vesktop)
- When a module needs unstable packages, import via `import inputs.nixpkgs-unstable { system = ...; config.allowUnfree = true; };`
- Never use `nixpkgs-unstable` as the default ‚Äî it's only for packages that don't exist or are too old on stable

### Scripts Directory Auto-Package

`scripts/*.sh` files are auto-converted to packages via `pkgs.writeShellScriptBin`. Any script placed in `scripts/` becomes available in the user's PATH. The `home.packages` module reads the directory and generates package derivations.

## Commands

| Action | Command |
|--------|---------|
| Switch on current machine | `nh switch` (nh is installed and configured for this flake) |
| Build for a specific host | `nixos-rebuild build --flake .#p1g3` or `.#t14s` |
| Deploy to a host | `sudo nixos-rebuild switch --flake .#p1g3` |
| Validate flake | `nix flake check` |
| Format nix files | `nix fmt` (uses nixfmt) |
| Dev shell | `nix develop` |
| Pre-commit hooks | `nix develop -c pre-commit run --all-files` |
| Update all inputs | `nix flake update` |
| Update single input | `nix flake update <name>` |

> **Note:** Use `nh` for daily rebuilds on the current machine. Use `nixos-rebuild --flake .#<host>` for cross-host builds or when targeting a specific host.

## Architecture Notes

This repo uses the Dendritic pattern (`flake-parts` + `import-tree`). The `dendritic-pattern` skill covers module registration, host assembly, adding features/hosts, and common pitfalls. Load it when working on any `.nix` file under `modules/`.
